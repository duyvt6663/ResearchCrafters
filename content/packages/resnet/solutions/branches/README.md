Each file here documents the *outcome* of a non-canonical branch from
stage S002. These notes are not served to the AI mentor before the
learner has completed the package: the package's stage policy sets
`branch_solutions: never` for stages S001-S007 and
`branch_solutions: after_completion` for the reflection stage S008.

Files:

- `branch-deeper-no-residual.md` — narrative + sketch showing why
  stacking more plain layers does not solve degradation.
- `branch-bottleneck-suboptimal.md` — narrative + sketch showing why
  bottleneck topology without an identity shortcut still suffers
  degradation.

The canonical branch (`branch-residual-canonical`) does not have a
sibling note here because its full solution is `solutions/canonical/`.
